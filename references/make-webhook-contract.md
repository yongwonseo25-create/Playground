# Make.com Webhook Contract for `POST /api/voice/submit`

## 1) Source of Truth

- Route: `src/app/api/voice/submit/route.ts`
- Sender: `src/server/reliability/WebhookClient.ts`
- Signer: `src/server/webhook/WebhookSigner.ts`

이 문서는 위 코드에서 실제로 전송되는 스펙만 기준으로 작성되었다.

## 2) Outbound Request (Voxera -> Make.com)

- Method: `POST`
- URL: `MAKE_WEBHOOK_URL` (or fallback `NEXT_PUBLIC_WEBHOOK_URL`)
- Content-Type: `application/json`
- Body: JSON 문자열 (`JSON.stringify(payload)`)

### Payload Schema (actual)

```json
{
  "clientRequestId": "string (required)",
  "transcriptText": "string (required)",
  "spreadsheetId": "string (optional, default: \"\")",
  "slackChannelId": "string (optional, default: \"\")",

  "audio_duration_sec": "number (optional, default: 0)",

}
```

## 3) Headers Generated by Voxera

Voxera가 Make.com Webhook으로 보낼 때 실제로 추가하는 헤더:

- `Content-Type: application/json`
- `X-Webhook-Timestamp: <ISO-8601 UTC timestamp>`
- `X-Webhook-Signature: <hex hmac-sha256>`
- `X-Idempotency-Key: <clientRequestId>`

### Signature Algorithm (exact)

- message: `<X-Webhook-Timestamp>.<raw_json_body>`
- digest: `HMAC-SHA256(secret=MAKE_WEBHOOK_SECRET, message)`
- encoding: lowercase hex

즉,

```text
signature = HMAC_SHA256_HEX(MAKE_WEBHOOK_SECRET, `${timestamp}.${body}`)
```

## 4) Make.com Scenario Verification Guide (production pattern)

권장 시나리오 순서:

1. `Webhooks > Custom webhook`
2. `Tools > Set multiple variables` (header/body 추출)
3. `Code` (JavaScript) module (HMAC 검증)
4. `Flow Control > Router` (검증 성공/실패 분기)
5. 성공 브랜치: `OpenAI (Chat Completions)` -> 후속 Slack/Sheets/Notion
6. 실패 브랜치: 로깅 후 종료 (또는 에러 핸들링)

### Step A. Webhook에서 받아야 하는 값

Custom webhook 수신값에서 다음을 추출:

- `timestamp = headers["x-webhook-timestamp"]`
- `signature = headers["x-webhook-signature"]`
- `idempotencyKey = headers["x-idempotency-key"]`
- `rawBody = raw request body string`

주의:

- 서명 검증은 "raw body 문자열" 기준이다.
- JSON 오브젝트를 다시 serialize하면 키 순서/공백 차이로 검증이 깨질 수 있다.

### Step B. Code (JavaScript) module 예시

입력 변수:

- `timestamp`
- `signature`
- `rawBody`
- `secret` (Make.com 연결 변수/시크릿 스토어에서 주입)

예시 코드:

```javascript
import crypto from 'crypto';

const timestamp = input.timestamp;
const providedSig = (input.signature || '').toLowerCase();
const rawBody = input.rawBody;
const secret = input.secret;

if (!timestamp || !providedSig || !rawBody || !secret) {
  throw new Error('Missing signature inputs');
}

const message = `${timestamp}.${rawBody}`;
const expectedSig = crypto.createHmac('sha256', secret).update(message).digest('hex');

const ok =
  expectedSig.length === providedSig.length &&
  crypto.timingSafeEqual(Buffer.from(expectedSig, 'hex'), Buffer.from(providedSig, 'hex'));

if (!ok) {
  throw new Error('Invalid webhook signature');
}

// optional anti-replay: 5-minute window
const driftMs = Math.abs(Date.now() - Date.parse(timestamp));
if (Number.isNaN(driftMs) || driftMs > 5 * 60 * 1000) {
  throw new Error('Stale webhook timestamp');
}

return {
  verified: true,
  payload: JSON.parse(rawBody),
  idempotencyKey: input.idempotencyKey || ''
};
```

### Step C. Duplicate Guard (recommended)

- `X-Idempotency-Key` (`clientRequestId`)를 Make Data Store 키로 사용.
- 이미 처리된 키면 라우터에서 종료.
- 신규 키면 저장 후 GPT-4o 모듈 진행.

## 5) GPT-4o Module Handoff (Make.com)

검증 성공 후 OpenAI 모듈 입력 매핑:

- model: `gpt-4o`
- user content: `payload.transcriptText`
- metadata 권장 첨부:
  - `payload.clientRequestId`
  - `payload.sessionId`
  - `payload.createdAt`
  - `payload.slackChannelId`
  - `payload.spreadsheetId`
  - `payload.notionDatabaseId`
  - `payload.notionParentPageId`
  - `payload.stt_provider`
  - `payload.audio_duration_sec`

권장 system prompt 예시:

```text
You are an execution assistant for Korean business voice notes.
Return concise actionable items in Korean.
```

## 6) Operational Notes

- Voxera는 전송 실패 시 재시도(최대 3회) + exponential backoff + circuit breaker를 사용한다.
- 최종 실패 시 내부 failure queue로 적재 후 재처리를 시도한다.
- 따라서 Make.com은 동일 `X-Idempotency-Key` 중복 수신을 전제로 설계해야 한다.
