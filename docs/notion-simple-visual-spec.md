# VOXERA Notion 실행 대시보드 시각 명세 1차안

이 문서는 **노션 실전 사용자 화면**을 기준으로 만든 1차 시각 명세입니다.
목표는 복잡한 위키가 아니라, 한국 초보 사용자도 바로 이해하는 **실행 중심 노션 대시보드**입니다.

## 핵심 원칙

- 첫 화면에서 바로 `검토 -> 실행 전환 -> 실행 관리` 흐름이 보여야 한다
- 설명문은 최소화하고, 실제 사용 요소만 보여야 한다
- `받은 음성함`은 검토용
- `실행 보드`는 실행용
- 긴 문서형 가이드는 메인에서 빼고, 필요한 경우 링크만 둔다

## 화면 구조

1. 상단 제목
- `VOXERA 실행 대시보드`

2. KPI 카드 4개
- 검토할 음성
- 진행 중 업무
- 오늘 마감
- 완료한 업무

3. 중단 메인 영역
- 왼쪽: `받은 음성함`
- 가운데: `실행 전환`
- 오른쪽: `실행 보드`

4. 하단 바로가기
- `받은 음성함 열기`
- `실행 보드 열기`
- `실행 가이드`

## 노션 요소 매핑

### 제목
- 일반 페이지 제목

### KPI 카드
- Callout 또는 synced block 카드형
- 숫자 크게, 라벨 작게

### 받은 음성함
- linked database view
- 노출 컬럼
  - 한 줄 요약
  - 말한 사람
  - 우선순위
  - 마감일
  - 상태

### 실행 보드
- linked database view
- 노출 컬럼
  - 할 일
  - 담당자
  - 상태
  - 우선순위
  - 마감일

### 실행 전환
- 실제 버튼처럼 보이지만 설명형 노드
- 시각적으로만 흐름을 연결

### 바로가기
- 페이지 링크 또는 버튼형 링크 블록

## 반드시 지킬 것

- 메인 화면에 긴 설명 문단 넣지 말 것
- 관계형/롤업/자동화는 기본 화면에 드러내지 말 것
- 메타 컬럼은 숨길 것
- 한눈에 `받은 음성함`과 `실행 보드`의 차이가 보여야 함

## 톤앤매너

- 밝은 파스텔톤
- 화이트 배경 + 얇은 파스텔 테두리
- 부드러운 그림자
- 둥근 카드
- 친절하지만 가벼운 느낌
- 위키보다 앱에 가까운 정돈된 구성

## 클릭 요소

- 상단 KPI 카드: 클릭되면 각 뷰로 이동 가능
- `받은 음성함 열기`: 받은 음성함 DB로 이동
- `실행 보드 열기`: 실행 보드 DB로 이동
- `실행 가이드`: 별도 가이드 페이지로 이동

## 비클릭 요소

- 가운데 `실행 전환`
- 카드 내부의 예시 필드 박스

## 구현 우선순위

1. 노션 시각 블루프린트 확정
2. 노션 대시보드 실제 블록 배치
3. KPI 카드/링크 블록 스타일 통일
4. 받은 음성함 / 실행 보드 linked database 정리
5. 메타 컬럼 숨김
