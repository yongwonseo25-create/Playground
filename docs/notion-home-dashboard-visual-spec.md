# VOXERA Notion 홈 대시보드 시각 명세 2차안

이 문서는 **노션 메인 페이지를 열자마자 바로 보이는 홈 대시보드**를 기준으로 만든 2차 시각 명세입니다.
구글 워크스페이스처럼 별도 모달을 여는 방식이 아니라, **노션 페이지 자체가 실행 홈 화면**이 되는 구조입니다.

## 핵심 원칙

- 페이지에 들어오면 바로 `검토 -> 실행 전환 -> 실행 관리` 흐름이 보여야 한다
- 긴 설명 문단은 넣지 않는다
- 실제 노션 블록처럼 보이는 요소만 남긴다
- `받은 음성함`은 검토용 linked database
- `실행 보드`는 실행용 linked database
- `실행 가이드`는 별도 링크 블록으로만 둔다

## 화면 구조

1. 페이지 제목
- `VOXERA 실행 대시보드`

2. 상단 KPI 카드 4개
- 검토할 음성
- 진행 중 업무
- 오늘 마감
- 완료한 업무

3. 중단 메인 2열
- 왼쪽: `받은 음성함`
- 오른쪽: `실행 보드`

4. 두 보드 사이 흐름 라벨
- `실행 전환`
- 노드형 표시만 두고, 버튼처럼 과하게 보이지 않게 한다

5. 하단 빠른 링크
- `받은 음성함 열기`
- `실행 보드 열기`
- `실행 가이드`

## 노션 요소 매핑

### 페이지 제목
- 일반 제목 블록

### KPI 카드
- callout 카드 4개 또는 synced block 카드 4개
- 숫자는 크게, 라벨은 짧게

### 받은 음성함
- linked database view
- 노출 컬럼
  - 한 줄 요약
  - 말한 사람
  - 우선순위
  - 마감일
  - 상태

### 실행 보드
- linked database view
- 노출 컬럼
  - 할 일
  - 담당자
  - 상태
  - 우선순위
  - 마감일

### 실행 전환
- 작은 흐름 배지 또는 callout
- 설명 텍스트는 넣지 않는다

### 하단 링크
- 버튼형 링크 블록 또는 페이지 링크 블록

## 반드시 지킬 것

- 홈 화면에 긴 가이드 본문을 넣지 않는다
- 메타 컬럼은 기본 뷰에서 숨긴다
- 관계형/롤업/자동화는 메인 화면에 드러내지 않는다
- 받는 공간과 실행 공간이 한눈에 구분돼야 한다
- 노션 페이지를 열면 추가 동작 없이 바로 보이는 구조여야 한다

## 톤앤매너

- 밝은 파스텔톤
- 화이트 배경
- 얇은 파스텔 테두리
- 둥근 카드
- 부드러운 그림자
- 위키보다 앱에 가까운 정돈된 느낌

## 클릭 요소

- KPI 카드: 각 필터 뷰로 이동 가능
- 받은 음성함 열기
- 실행 보드 열기
- 실행 가이드

## 비클릭 요소

- 실행 전환 배지
- 데이터 미리보기 카드

## 구현 우선순위

1. 시각 블루프린트 확정
2. 노션 페이지 블록 배치
3. KPI 카드 스타일 정리
4. 받은 음성함 / 실행 보드 linked database 뷰 정리
5. 하단 빠른 링크 배치
